# 完成イメージ図解を作成するためのプロンプト

以下を、AI画像生成ツールまたはデザイナにそのまま貼り付けてください。出力は**SVG**（推奨）または**PNG**。日本語ラベルは**UTF-8**で保存し、文字化けを防ぐために、ツール側で文字エンコーディングをUTF-8に固定してください。

---

## 役割
あなたは技術ドキュメント用の図解デザイナです。プロジェクト**Aviation-News**の、完成後の姿（データの流れ＋公開Webの見た目）を**1枚の横長イメージ図**（約1000×760px前後）でまとめてください。

## プロジェクト概要
- 名前: **Aviation-News**
- Aviation Wire の **RSS** を取得し、`feeds.yaml` のキーワードで **JAL系 / ANA系** に分類して HTML レポートを自動生成する。
- **GitHub Actions** で定期実行し、成果物を **GitHub Pages** で公開する。
- 公開URLの例: `https://<user>.github.io/Aviation-News/`

## 図の構成（必順2ブロック）

### ブロック1: データの流れ（上部）
左から右へ矢印でつなぐフローチャート。ボックスの順と表記は次の通り。

1. `feeds.yaml` （キーワード定義）
2. `fetcher.py` （RSS取得・絞り込み）
3. `items.json` （中間データ）
4. `renderer.py` （HTML生成）
5. `public/` （`index.html` ほか）
6. `GitHub Actions` （毎日ビルド）
7. `Pages` （公開URL）

見出し文字例: **1. データの流れ（CI とローカルで同じ）**

### ブロック2: 公開ページの見た目（下部）
簡易なブラウザウィンドウ（アドレスバー、ウィンドウボタン）の中に、実際のWebページのワイヤーフレームを描く。

- ページタイトル: **JAL / ANA 関連ニュース**
- サブ文例: **データソース: Aviation Wire（RSS） · 生成: YYYY-MM-DD HH:MM JST**
- **2カラム**:
  - **左: JAL・グループ** — 見出し色 **#c40018**、背景は薄いピンク系（例: #fff5f5）。サブ: 「日本航空 / ZIPAIR / JTA・RAC・JAC など」
  - **右: ANA・グループ** — 見出し色 **#003894**、背景は薄い青系（例: #f3f7fc）。サブ: 「全日空 / ANA HD など」
- 各カラムに記事カードを2件程度: リンク色のタイトル、日付・ソース行、要約の1〜2行。
- 片方に記事がない場合は **該当なし（プレースホルダー）** のような灰色説明でよい。

見出し文字例: **2. 公開ページの見た目（2カラム対比）**

## スタイル
- 言語: 日本語ラベル（UI文言）
- トーン: 落ち着いたプロダクト紹介向け。装飾過多にしない。
- 全体背景は極めて薄いグレー系のグラデーション。
- ボックスは白背景、細い枡線、軽いドロップシャドウ。
- フッターに小さく: 「両社にまたがる記事は上段『両社・グループに関連』に表示される」という注記（レイアウト上は省略可）。

## 制約・文字化け対策（重要）
- 出力ファイルは **UTF-8**（BOMなしで可）。
- SVG の場合: 日本語は **XML数値文字参照（&#xXXXX;）** で記述すると、ツール間での文字化けに強い（推奨）。
- フォントは **Meiryo / Yu Gothic / Noto Sans CJK JP** 等をフォントスタックに含め、欠けグリフ（豆腐）を防ぐ。
- 円囲数字①②などに依存しすぎない（必要なら **1.** **2.** などASCII番号）。
- 公式ロゴマークは使わない（テキストと色で連想させる程度）。

## 成果物
- 印刷・スライド・README嵌めに耐えるコントラストと解像度。
- 可能ならテキストはアウトライン化しすぎず、編集可能なまま。
